## 🤖 myCobot 280 + 摄像头 实战复盘

### 一、环境搭建

| 项目 | 详情 |
|------|------|
| **机械臂** | myCobot 280，串口 `/dev/cu.usbserial-140` |
| **摄像头** | USB 摄像头，OpenCV index 0 |
| **波特率** | 1000000（不是默认的 115200！） |
| **Python** | 3.12（终端）+ 3.14（OpenClaw） |
| **关键库** | pymycobot 4.0.4、opencv-python、Pillow |

### 二、踩坑记录

**坑1：摄像头权限**
- OpenClaw 的 Node.js 进程没有 macOS 摄像头权限
- `cv2.VideoCapture()` 会卡死或返回空帧
- **解决方案**：通过 `osascript` 调用 Terminal.app 执行 Python 脚本，继承终端的摄像头权限
```bash
osascript -e 'tell application "Terminal" to do script "python3 your_script.py"'
```

**坑2：摄像头首帧发绿**
- USB 摄像头刚打开时前几帧是坏的（YUYV 格式解码错误）
- **解决方案**：
  - 设置 MJPEG 格式：`cap.set(cv2.CAP_PROP_FOURCC, cv2.VideoWriter_fourcc(*'MJPG'))`
  - 跳过前 10 帧预热

**坑3：8080 端口被占**
- OpenClaw 自己占用了 8080 端口
- 视频流服务器改用 8888 端口

**坑4：MediaPipe 新版 API 变化**
- mediapipe 0.10+ 去掉了 `mp.solutions`，需用 Task API
- 改用 OpenCV 肤色检测 + 凸包缺陷分析做手势识别

### 四、飞书发图经验

| 格式 | 效果 | 大小建议 |
|------|------|----------|
| **JPG/PNG** | ✅ 直接发图片，飞书正常显示 | 正常即可 |
| **GIF（小）** | ✅ 飞书渲染为动图，效果最好 | **≤3MB** |
| **GIF（大）** | ⚠️ 上传失败，变成文本链接 | 压缩后发 |
| **MP4** | ✅ 作为视频发送，可播放 | 正常即可 |

**GIF 压缩命令**：
```bash
ffmpeg -y -i input.gif -vf "fps=8,scale=320:-1:flags=lanczos" output.gif
```

**发图用 message 工具**：
```python
message(action="send", channel="feishu", filePath="xxx.gif", message="描述")
```
